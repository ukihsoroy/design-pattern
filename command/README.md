# 命令模式

## 0x01.定义与类型

- 定义：将“请求”封装成对象，以便使用不同的请求。
- 命令模式解决了应用程序中对象的职责以及它们之间的通信方式
- 类型：行为型

## 0x02.适用场景

- 请求调用者和请求接收者需要解耦，使得调用者和接收者不直接交互
- 需要抽象出等待执行的行为

## 0x03.优点

- 降低耦合
- 容易扩展新命令或一组命令

## 0x04.缺点

- 命令的无限扩展会增加类的数量，提高系统实现复杂度

## 0x05.相关设计模式

- 命令模式和备忘录模式
    - 适用备忘录模式保存命令的历史记录
